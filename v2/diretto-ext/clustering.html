diretto Clustering API

Introduction

Version

v2 - Draft

Author

Benjamin Erb

Description

The Clustering API is provided by an external service that extracts additional information from the collected documents and user-provided data. Actual implementations can choose different operational modes, such as live updates, asynchronous/delayed updates or periodic recomputations. Thus, services are not forced to return up-to-date information in any case, although services are encouraged to update data as soon as possible. Delayed or periodic updates will result in a eventually consistent data model. Service implementations also have free choice in selecting features and metrics for calculating similarity, relations, trends etc.
